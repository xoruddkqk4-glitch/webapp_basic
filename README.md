# 📋 DID 게시물 신청 시스템

이 웹앱은 **디지털 게시판(DID) 게시물의 게시 기간을 관리하고 시각화하는 시스템**입니다.

---

## 🎯 핵심 기능

### 1. 게시물 등록 기능

**입력 항목:**
- 부서 (필수)
- 담당자 (필수)
- 제목 (필수)
- 게시 시작일 (필수)
- 게시 종료일 (필수)
- 파일 첨부 (선택, 최대 2MB)

**주요 특징:**
- **데이터 저장:** Firebase Firestore에 저장
- **파일 업로드:** Firebase Storage에 첨부파일 업로드 및 다운로드 링크 생성
- **이메일 알림:** EmailJS를 통해 게시물 등록 시 관리자에게 이메일 발송

### 2. 게시 기간 차트 (Gantt Chart)

**시각화 기능:**
- 각 게시물의 게시 기간을 간트 차트로 표시
- 오늘 날짜를 빨간색 수직선으로 표시
- 드래그 앤 드롭으로 순서 변경 가능

**게시 상태별 색상:**
- 🟢 **게시 중** (active): 녹색 그라데이션
- 🔵 **게시 예정** (pending): 파란색 그라데이션
- ⚫ **게시 종료** (expired): 회색 그라데이션

### 3. 게시물 목록 관리

**테이블 형식 표시:**
- 부서, 담당자, 제목, 시작일, 종료일, 상태, 첨부파일, 삭제 버튼

**정렬 기능:**
- 시작일 기준 오름차순/내림차순
- 종료일 기준 오름차순/내림차순

**추가 기능:**
- 드래그 앤 드롭으로 항목 순서 변경
- 게시 상태 실시간 표시 (게시 중/예정/종료)

### 4. 파일 관리

- **첨부파일 다운로드:** 각 게시물의 첨부파일 다운로드 링크 제공
- **파일 크기 제한:** 최대 2MB
- **자동 삭제:** 게시물 삭제 시 첨부파일도 Firebase Storage에서 자동 삭제

### 5. 삭제 기능

- **개별 삭제:** 각 게시물마다 삭제 버튼 제공
- **전체 삭제:** "전체 삭제" 버튼으로 모든 게시물 일괄 삭제
- **이메일 알림:** 게시물 삭제 시에도 관리자에게 이메일 발송
- **확인 메시지:** 삭제 전 확인 팝업 표시

## 🛠️ 기술 스택

### 백엔드/데이터베이스
- **Firebase Firestore:** 게시물 데이터 저장 (실시간 클라우드 데이터베이스)
- **Firebase Storage:** 첨부파일 저장
- **서버 타임스탬프:** 게시물 생성 시간 자동 기록

### 알림 시스템
- **EmailJS:** 게시물 등록/삭제 시 이메일 알림
- **알림 내용:** 부서, 담당자, 제목, 작업 시간 등

### 시간대 처리
- **서울 시간대 기준:** 모든 날짜 처리를 Asia/Seoul 시간대로 통일
- **UTC 변환:** 정확한 날짜 비교를 위해 UTC 기준으로 변환하여 처리

### UI/UX
- **반응형 디자인:** 모바일, 태블릿, 데스크톱 화면에 대응
- **드래그 앤 드롭:** 게시물 순서 재배치 가능
- **실시간 상태 표시:** 오늘 날짜를 기준으로 게시 상태 자동 계산
- **모던한 디자인:** 그라데이션, 그림자, 부드러운 애니메이션 효과

## 💡 주요 사용 시나리오

### 1. 담당자가 새 게시물 등록
폼에 정보 입력 → 제출 → Firestore에 저장 → 관리자에게 이메일 발송

### 2. 게시 기간 시각화
간트 차트에서 모든 게시물의 게시 기간을 한눈에 확인  
오늘 날짜 기준으로 진행 상황 파악

### 3. 게시물 관리
- 목록에서 날짜별 정렬
- 드래그로 순서 변경
- 불필요한 게시물 삭제

### 4. 파일 다운로드
각 게시물의 첨부파일 다운로드
